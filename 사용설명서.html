2. 기술 스택

항목내용

3. 사용 방법 (참가자)

로그인

  1. 사이트 접속 시 닉네임 입력 팝업이 표시됩니다.
  2. 원하는 닉네임을 입력하고 시작하기 버튼을 클릭합니다.
  3. 같은 닉네임은 어느 기기·브라우저에서 접속해도 동일한 계정으로 인식됩니다.
  4. 로그인 정보는 브라우저 로컬스토리지에 저장되어 재접속 시 자동 로그인됩니다.
주의: 닉네임은 비밀번호 없이 입력만으로 로그인됩니다. 본인만 아는 고유한 닉네임 사용을 권장합니다.

작품 업로드

  1. 상단 메뉴 업로드 탭 클릭
  2. 작품 제목 입력 (선택사항)
  3. 이미지 파일 선택 (JPG, PNG, WEBP 지원)
  4. 작품 업로드하기 버튼 클릭
  5. 업로드된 작품은 갤러리 탭에서 확인 가능합니다.

투표

  1. 갤러리 탭에서 작품 목록 확인
  2. 마음에 드는 작품의 하트(♥) 버튼 클릭
  3. "이 작품에 투표하시겠습니까?" 확인 팝업이 표시됩니다.
  4. 투표하기 버튼을 눌러 최종 확정합니다. (취소 가능)
  5. 1인당 최대 2표 행사 가능 (다른 작품에 각 1표씩)
  6. 본인이 업로드한 작품에는 투표 불가
  7. 이미 투표한 작품은 버튼이 비활성화됩니다.
투표는 취소되지 않습니다. 팝업에서 신중하게 확인 후 투표하세요.

결과 확인

  • 결과 탭: 관리자가 투표를 종료한 후 최종 순위 공개

자료 다운로드

  • 자료 탭: 관리자가 업로드한 강의 교안 등 파일 다운로드

4. 사용 방법 (관리자)

관리자 패널 접근

  1. 상단 메뉴 관리 탭 클릭
  2. 관리자 비밀번호 입력 후 인증

주요 기능

기능설명
투표 종료투표를 마감하고 결과 탭에 최종 순위 공개
비밀번호 변경관리자 비밀번호 변경
데이터 백업전체 데이터 JSON 형식으로 다운로드
로그인 기록 조회로그인한 닉네임 전체 목록 및 투표 사용 현황 확인
작품 삭제부적절한 작품 개별 삭제
자료 업로드강의 교안 등 파일 업로드
자료 삭제업로드된 자료 삭제
데이터 초기화전체 작품·투표 데이터 리셋 (재사용 시)
초기 관리자 비밀번호: admin1234 (최초 접속 후 반드시 변경하세요)

로그인 기록 조회 방법

  1. 관리자 패널 인증 후
  2. "목록 불러오기" 버튼 클릭
  3. 로그인한 닉네임 목록과 각 닉네임의 투표 사용 현황(0/2, 1/2, 2/2)이 표시됩니다.

5. 보안 구조

중복 투표 방지

  • 닉네임 기반 세션 토큰(UUID)을 서버에 저장
  • 동일 닉네임 = 동일 세션 → 투표 2회 한도 공유
  • 브라우저/기기를 바꿔도 같은 닉네임이면 추가 투표 불가

자기 투표 방지

  • 작품 업로드 시 업로더 닉네임을 서버에 저장
  • 투표 요청 시 세션 닉네임과 업로더 닉네임 비교 → 일치하면 거부

투표 실수 방지

  • 하트 버튼 클릭 시 확인 팝업 표시
  • 최종 "투표하기" 버튼을 눌러야만 투표 확정

속도 제한 (Rate Limiting)

엔드포인트제한
로그인IP당 20회 / 30초
투표IP당 20회 / 30초
업로드IP당 10회 / 30초
관리자IP당 20회 / 60초

비밀번호 보안

  • 관리자 비밀번호는 SHA-256 + 랜덤 Salt 해싱 후 저장
  • 평문 비밀번호는 서버에 저장되지 않음

6. 데이터 저장 구조

로컬 파일

wallpaper-contest/
├── data.json          # 작품 목록, 투표 데이터, 세션 정보
├── public/
│   ├── uploads/       # 업로드된 작품 이미지
│   └── docs/          # 관리자 업로드 자료

data.json 구조

{
  "works": [
    {
      "id": "uuid",
      "title": "작품 제목",
      "filename": "이미지파일명.jpg",
      "votes": 3,
      "uploaderNickname": "닉네임",
      "uploadedAt": "2025-01-01T00:00:00"
    }
  ],
  "sessions": {
    "닉네임": {
      "sessionToken": "uuid",
      "votes": ["작품id1", "작품id2"]
    }
  },
  "adminPassword": "sha256:salt:hash",
  "votingEnded": false
}

Cloudflare R2 백업

  • 서버가 재시작되면 R2에서 data.json + 이미지 + 자료 전체를 자동 복구
  • Railway 무료 플랜의 컨테이너 재시작으로 인한 데이터 소실 방지
  • R2 무료 한도: 저장 10GB / 쓰기 100만 건/월 / 읽기 1,000만 건/월 / 데이터 전송 무료

7. 주의사항

  • 닉네임 관리: 비밀번호가 없으므로 닉네임이 곧 계정입니다. 고유한 닉네임 사용을 권장합니다.
  • 투표 확정 후 취소 불가: 팝업에서 "투표하기"를 누르면 취소할 수 없습니다. 신중하게 선택하세요.
  • 이미지 형식: JPG, PNG, WEBP만 업로드 가능합니다.
  • 관리자 비밀번호: 초기값 admin1234에서 반드시 변경하세요.
  • 데이터 초기화: 관리자 패널의 초기화 기능은 되돌릴 수 없습니다.
  • Railway 재배포: GitHub에 push하면 약 1~2분간 서비스가 재시작됩니다.

본 서비스는 사내 공모전 전용으로 제작되었습니다.
