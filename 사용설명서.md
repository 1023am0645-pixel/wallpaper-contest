## 1. 서비스 개요

| 항목 | 내용 |

## 2. 기술 스택

### 백엔드
| 항목 | 내용 |
|---|---|
| 언어 | Python 3.11 |
| 서버 | Python 표준 라이브러리 `http.server.HTTPServer` (외부 프레임워크 없음) |
| 인증 | 닉네임 기반 세션 토큰 (UUID v4) |
| 비밀번호 | SHA-256 + 랜덤 Salt 해싱 (`hashlib`) |
| 동시성 | `threading.Lock` 으로 동시 쓰기 충돌 방지 |
| 속도 제한 | IP 기반 커스텀 Rate Limiter (투표 20회/분, 로그인 20회/분) |

### 프론트엔드
| 항목 | 내용 |
|---|---|
| 구성 | 단일 HTML 파일 (`public/index.html`) |
| 아이콘 | Lucide Icons (CDN) |
| 테마 | 다크 / 화이트 / 벚꽃 3가지 지원 |
| 통신 | Fetch API (REST JSON) |

### 스토리지
| 항목 | 내용 |
|---|---|
| 주 저장소 | 서버 로컬 파일시스템 (`data.json`, `uploads/`, `docs/`) |
| 클라우드 백업 | Cloudflare R2 (S3 호환 오브젝트 스토리지) |
| 동기화 방식 | Write-through: 저장 즉시 R2 비동기 백업 / 서버 시작 시 R2→로컬 복구 |
| SDK | `boto3` (AWS S3 호환) |

### 배포
| 항목 | 내용 |
|---|---|
| 플랫폼 | Railway (PaaS) |
| 연동 | GitHub 레포지토리 push → 자동 재배포 |
| 실행 명령 | `python3 server.py` (`Procfile` 정의) |

## 3. 사용 방법 (참가자)

### 로그인
1. 사이트 접속 시 닉네임 입력 팝업이 표시됩니다.
2. 원하는 닉네임을 입력하고 **입장** 버튼을 클릭합니다.
3. 같은 닉네임은 어느 기기·브라우저에서 접속해도 동일한 계정으로 인식됩니다.
4. 로그인 정보는 브라우저 로컬스토리지에 저장되어 재접속 시 자동 로그인됩니다.

> **주의**: 닉네임은 비밀번호 없이 입력만으로 로그인됩니다. 다른 사람이 동일 닉네임을 입력하면 같은 계정으로 접속됩니다. 본인만 아는 고유한 닉네임 사용을 권장합니다.

### 작품 업로드
1. 상단 메뉴 **업로드** 탭 클릭
2. 작품 제목 입력 (선택사항)
3. 이미지 파일 선택 (JPG, PNG, WEBP 지원)
4. **작품 업로드하기** 버튼 클릭
5. 업로드된 작품은 **갤러리** 탭에서 확인 가능합니다.

### 투표
1. **갤러리** 탭에서 작품 목록 확인
2. 마음에 드는 작품의 **투표** 버튼 클릭
3. 1인당 최대 **2표** 행사 가능 (다른 작품에 각 1표씩)
4. 본인이 업로드한 작품에는 투표 불가
5. 이미 투표한 작품은 버튼이 비활성화됩니다.
6. 투표 취소는 불가합니다.

### 결과 확인
- **결과** 탭: 관리자가 투표를 종료한 후 최종 순위 공개

### 자료 다운로드
- **자료** 탭: 관리자가 업로드한 강의 교안 등 파일 다운로드

## 4. 사용 방법 (관리자)

### 관리자 패널 접근
1. 상단 메뉴 **관리** 탭 클릭
2. 관리자 비밀번호 입력 후 인증

### 주요 기능

| 기능 | 설명 |
|---|---|
| 투표 종료 | 투표를 마감하고 결과 탭에 최종 순위 공개 |
| 작품 삭제 | 부적절한 작품 개별 삭제 |
| 자료 업로드 | 강의 교안 등 파일 업로드 (참가자가 자료 탭에서 다운로드 가능) |
| 자료 삭제 | 업로드된 자료 삭제 |
| 데이터 초기화 | 전체 작품·투표 데이터 리셋 (재사용 시) |
| 비밀번호 변경 | 관리자 비밀번호 변경 |
| 데이터 내보내기 | 전체 데이터 JSON 형식으로 다운로드 |

> **초기 관리자 비밀번호**: `admin1234` (최초 접속 후 반드시 변경하세요)

## 5. 보안 구조

### 중복 투표 방지
- 닉네임 기반 세션 토큰(UUID)을 서버에 저장
- 동일 닉네임 = 동일 세션 → 투표 2회 한도 공유
- 브라우저/기기를 바꿔도 같은 닉네임이면 추가 투표 불가

### 자기 투표 방지
- 작품 업로드 시 업로더 닉네임을 서버에 저장
- 투표 요청 시 세션 닉네임과 업로더 닉네임 비교 → 일치하면 거부

### 속도 제한 (Rate Limiting)
| 엔드포인트 | 제한 |
|---|---|
| 로그인 | IP당 20회/분 |
| 투표 | IP당 20회/분 |
| 업로드 | IP당 10회/분 |
| 관리자 | IP당 40회/15분 |

### 비밀번호 보안
- 관리자 비밀번호는 SHA-256 + 랜덤 Salt 해싱 후 저장
- 평문 비밀번호는 서버에 저장되지 않음

## 6. 데이터 저장 구조

### 로컬 파일
```
wallpaper-contest/
├── data.json          # 작품 목록, 투표 데이터, 세션 정보
├── public/
│   ├── uploads/       # 업로드된 작품 이미지
│   └── docs/          # 관리자 업로드 자료
```

### data.json 구조
```json
{
  "works": [
    {
      "id": "uuid",
      "title": "작품 제목",
      "filename": "이미지파일명.jpg",
      "votes": 3,
      "uploaderNickname": "닉네임",
      "uploadedAt": "2025-01-01T00:00:00"
    }
  ],
  "sessions": {
    "닉네임": {
      "sessionToken": "uuid",
      "votes": ["작품id1", "작품id2"]
    }
  },
  "adminPassword": "sha256:salt:hash",
  "votingEnded": false
}
```

### Cloudflare R2 백업
- 서버가 재시작되면 R2에서 data.json + 이미지 + 자료 전체를 자동 복구
- Railway 무료 플랜의 컨테이너 재시작으로 인한 데이터 소실 방지
- R2 무료 한도: 저장 10GB / 쓰기 100만 건/월 / 읽기 1,000만 건/월 / 데이터 전송 무료

## 7. 주의사항

- **닉네임 관리**: 비밀번호가 없으므로 닉네임이 곧 계정입니다. 고유한 닉네임 사용을 권장합니다.
- **이미지 형식**: JPG, PNG, WEBP만 업로드 가능합니다.
- **투표 취소 불가**: 한 번 투표한 작품은 취소할 수 없습니다.
- **관리자 비밀번호**: 초기값 `admin1234`에서 반드시 변경하세요.
- **데이터 초기화**: 관리자 패널의 초기화 기능은 되돌릴 수 없습니다.
- **Railway 재배포**: GitHub에 push하면 약 1~2분간 서비스가 재시작됩니다. 이 시간 동안은 접속이 잠시 끊길 수 있습니다.
